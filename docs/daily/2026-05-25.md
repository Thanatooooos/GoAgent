# 2026-05-25 工作记录

## 结构收口 + Memory Phase 4 收尾

### 今日目标

今天这轮工作分成两条主线：

1. 把此前已经明确的结构重构真正落到代码上，优先收掉 `Knowledge` 服务层和 `HTTP` 入口层的大文件职责混杂问题
2. 继续完成 `memory` 的 `Phase 4` 收尾，把 lifecycle maintenance 和 metrics 从“待办项”推进到“已落地能力”

今天没有开启新的产品 phase，也没有开始“自动从每轮消息学习用户偏好/习惯”。重点仍然是让当前主线更稳、更清晰、更可运维。

### 完成内容

#### 1. `internal/app/knowledge/service` 第一刀文件级重构落地

- `KnowledgeDocumentService` 拆成：
  - command
  - query
  - ingestion completion
  - builders
  - input helpers
- `KnowledgeChunkService` 拆成：
  - query
  - command
  - vector sync
  - support helpers
- `DocumentProcessService` 拆成：
  - public service
  - pipeline
  - persistence
  - runtime state

这一步保持了：

- 包边界不变
- 公开接口不变
- runtime 装配顺序不变

目标不是改业务语义，而是把过载文件先拆回可维护状态。

#### 2. `knowledge` 与 `rag` 的 HTTP handler 第二刀拆分落地

`internal/adapter/http/knowledge`

- 文档 handler 拆成 command/query/runtime/routes/views/form helpers

`internal/adapter/http/rag`

- 顶层 `handlers.go` 继续瘦身
- conversation/chat/feedback/memory/routes/shared/view types 分文件

这一轮的重点是把“单文件 400~500+ 行、多类接口混放”的问题收掉，避免 service 层刚收完、HTTP 层又重新堆复杂度。

#### 3. `longtermmemory/recall` 直接单测补了一轮

新增了围绕 `recall` 子包本身的直接测试，覆盖了：

- tokens
- ranking
- context renderer
- projection
- cache keys
- cache mappers

这一步的意义是把原来更多依赖 `memory_service_test.go` 间接覆盖的内容，下沉到 `recall` 子包自身，降低后续继续重构时的测试脆弱性。

#### 4. `memory` 的 lifecycle maintenance 服务能力落地

新增并接通了：

- `RunMaintenance(...)`
- 批量软过期已到 `expires_at` 的 memory
- 批量清理旧的 `expired / superseded` 记录
- repo 侧批量过期 / 批量清理契约与 SQL 骨架

这一步先把维护能力建起来，为后续 runtime 调度做基础闭环。

#### 5. `memory` Phase 4 收尾：后台 maintenance loop + metrics 补强落地

今天的关键收尾在这里。

已完成：

- `rag.memory.maintenance.*` 配置边界落地
  - `enabled`
  - `scan-delay-ms`
  - `run-timeout-ms`
  - `expire-batch-size`
  - `delete-batch-size`
  - `delete-retention-days`
- `internal/bootstrap/rag/runtime.go` 新增 memory maintenance loop
  - immediate first run
  - ticker 周期执行
  - per-run timeout
  - panic recover
  - `Close()` 时优雅停止
- `GET /rag/memory/metrics` 在保持兼容的前提下新增指标：
  - `maintenanceRuns`
  - `maintenanceFailures`
  - `maintenanceExpiredCount`
  - `maintenanceDeletedCount`
  - `embeddingGenerationFailures`
  - `embeddingPersistFailures`
  - `touchLastUsedFailures`
  - `scopeVersionLookupFailures`

也就是说，`memory` 这块现在已经不是只有 cache metrics，而是开始具备第一版运维观测面了。
