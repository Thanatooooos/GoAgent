## 三、goagent 尚未具备的 ragent 能力

### 1. 意图识别系统 ❌ 完全缺失

**ragent 具备**:
- 树形多级意图分类（DOMAIN → CATEGORY → TOPIC）
- LLM 意图识别 + 置信度判断
- 意图节点可视化编辑
- 意图与知识库/MCP 工具绑定
- 低置信度主动引导澄清

**goagent 现状**: 
- 无意图识别模块
- 所有请求走同一 RAG 链路
- 无法区分"查知识库"和"调业务工具"

**影响**: 无法实现检索与工具调用的智能路由

---

### 2. MCP 协议集成 ❌ 完全缺失

**ragent 具备**:
- MCP Server 独立进程
- MCP 工具自动发现与注册
- MCP 参数提取（LLM 提参）
- 非知识类意图自动调用业务工具
- 检索与工具无缝融合

**goagent 现状**:
- 无 MCP 协议支持
- Tool 是自研抽象，非标准协议
- 只能调用内置诊断 tool

**影响**: 无法对接外部业务系统工具

---

### 3. 多路检索 ❌ 完全缺失

**ragent 具备**:
- 多通道并行检索（向量 + 关键词 + 标题 + metadata）
- 后处理流水线（去重、重排、融合）
- 检索策略可插拔

**goagent 现状**:
- 仅有向量检索
- 支持 semantic/keyword/hybrid/auto 模式
- 但 hybrid 实际是语义层面的混合，非多路召回

**影响**: 对精确匹配（订单号、接口名）场景召回率偏低

---

### 4. 队列式并发限流 ❌ 完全缺失

**ragent 具备**:
- 基于 Redis 信号 + ZSET 的分布式排队限流
- Lua 脚本原子判断
- Pub/Sub 跨实例通知
- SSE 推送排队状态

**goagent 现状**:
- 有 rate limit 配置
- 但未见实际中间件实现
- 无排队机制

**影响**: 高并发时可能直接拒绝请求，而非排队等待

---

### 5. 首包探测机制 ❌ 完全缺失

**ragent 具备**:
- 模型切换时缓冲所有事件
- 确保用户端不收到半截脏数据
- 装饰器模式实现

**goagent 现状**:
- 模型切换直接中断当前流
- 无缓冲机制

**影响**: 模型 fallback 时用户体验可能不流畅

---

### 6. 管理后台仪表盘 ❌ 完全缺失

**ragent 具备**:
- 系统概览仪表盘
- 模型健康状态展示
- 请求量/延迟统计
- 反馈分布统计

**goagent 现状**:
- 无仪表盘页面
- 无统计可视化

**影响**: 运维监控能力弱

---

### 7. 样本问题管理 ❌ 完全缺失

**ragent 具备**:
- 样本问题配置
- 用于意图分类训练
- 前端展示示例问题

**goagent 现状**:
- 无样本问题管理

**影响**: 无法做效果评测基线

---

### 8. Query-Term 映射 ❌ 完全缺失

**ragent 具备**:
- Query 术语映射管理
- 用于检索增强

**goagent 现状**:
- 无此功能

**影响**: 专业术语检索效果可能不佳

## 四、goagent 独有而 ragent 没有的能力

### 1. 诊断能力 ✅ goagent 独有

- `document_ingestion_diagnose`: 文档入库失败诊断
- `task_ingestion_diagnose`: Task 执行失败诊断
- `trace_retrieval_diagnose`: 检索失败诊断
- 结论/证据/建议结构化输出

### 2. Tool 调用链 Trace ✅ goagent 独有

- 每次 tool call 独立写入 `rag_trace_node`
- 父子关系、耗时、状态完整记录
- 前端琥珀色卡片展示

### 3. WithoutCancel 上下文管理 ✅ goagent 独有

- 资源清理使用 `context.WithoutCancel`
- 更精细的上下文生命周期控制

## 六、建议优先级

### 第一阶段（P0 - 核心补齐）

1. **多路检索**: 提升召回率，对精确匹配场景更友好
2. **意图识别**: 实现请求智能路由，区分知识库查询和工具调用

### 第二阶段（P1 - 能力增强）

3. **MCP 协议**: 对接外部业务系统
4. **队列限流**: 提升高并发稳定性
5. **诊断能力增强**: 提升准确率和可信度

### 第三阶段（P2 - 体验优化）

6. **首包探测**: 优化模型切换体验
7. **仪表盘**: 运维可视化
8. **样本管理**: 效果评测

---

## 七、结论

goagent 已经具备了 ragent 约 **70% 的核心能力**，在 Knowledge、RAG 对话、模型路由等基础模块上已经与 ragent 对等。

**主要差距**集中在：
1. 意图识别系统（决定请求路由）
2. MCP 协议集成（对接外部工具）
3. 多路检索（提升召回率）

**独特优势**：
1. 诊断能力（ragent 没有）
2. Tool 调用链 Trace（ragent 没有）
3. 更精细的上下文管理

goagent 作为 Go 语言实现，在并发性能和资源占用上有天然优势，但在 AI 生态集成（如 MCP）上需要额外投入。
